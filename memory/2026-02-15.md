## Bug Discovery & Fix
**Issue**: Script `pintarblog-daily/index.js` tried to use `openclaw sessions spawn` command which doesn't exist in CLI
**Root cause**: `sessions_spawn` is only available as an AI agent tool, not a CLI command
**Error encountered**:
```
/bin/sh: 1: yaml: not found
/bin/sh: 2: ---: not found
```
(Special markdown characters in shell command being interpreted as redirects)

**Solution**: Rewrote cron job payload to directly generate blog posts via agentTurn, bypassing the Node.js script entirely

## Technical Lesson
- Don't use `openclaw sessions spawn` in shell scripts - command doesn't exist
- Use `agentTurn` in cron jobs for direct task execution
- For complex workflows, let the agent handle everything instead of trying to spawn sub-agents via CLI
