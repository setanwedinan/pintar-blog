# 2026-02-17

## USDIDR Blog Post Quality Issue & Fix

**Problem:** Faizal noticed USDIDR blog posts had placeholder text "Data tidak tersedia (memerlukan akses web search)" instead of actual market data like DXY, Asian currencies, etc.

**Root Cause:** USDIDR report is a Node.js script (`skills/usdidr-report/index.js`) that can't access web search tools. Unlike AI Bubble Dashboard which runs as an agent with full tool access.

**Solution Applied:**
1. **Updated cron job** (`7241a222-8ecb-4942-94a7-b5a0cd3289f3`):
   - Changed payload to have agent perform web search after running the script
   - Agent now researches: DXY, USD vs Asian currencies (JPY, CNY, SGD, THB, MYR, KRW), Indonesia foreign capital inflow/outflow
   - Agent generates comprehensive blog post with all data, replacing the placeholder file

2. **Fixed template literal bug** in script:
   - Prediction section had `${Math.round(result.rate * 1.01)}` appearing literally
   - Fixed by pre-calculating values before template string

3. **Tested live execution** (same day):
   - Ran script to get BCA data: 16,812
   - Web searched for DXY: 97.07 (down 2.34% monthly)
   - Web searched for Asian currencies:
     - USD/JPY: 153.18 (down 0.19%, JPY up 3.12% monthly)
     - USD/CNY: 6.91
     - USD/SGD: 1.2626
     - USD/THB: 31.13
     - USD/MYR: 3.905 (down 0.06%, MYR up 3.70% monthly)
     - USD/KRW: 1,463.70
   - Web searched for Indonesia capital flows: MSCI warning triggered capital flight, IHSG down 16.7% in 2 days
   - Generated comprehensive blog post with all data
   - Git committed & pushed: df86ea7
   - Reported to Telegram with link

**Key Learning:**
- Script-based skills can't use tools like web_search
- Agent-based skills (via cron agentTurn) have full tool access
- For comprehensive market analysis, run as agent not script
- Template literals must be pre-calculated before markdown generation
